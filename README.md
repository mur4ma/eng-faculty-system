# ระบบสารสนเทศคณะวิศวกรรมศาสตร์และเทคโนโลยี

เว็บแอปครบวงจร: ระบบสมาชิก 5 บทบาท, เก็บเอกสารนักศึกษา, เทียบโอน (ACC), หน้าหลักสูตร, รายวิชาเปิดสอน + workflow อนุมัติ, ลงทะเบียนแบบติ๊กเลือก + ตรวจเวลาชน, ตารางสอน, แจ้งเตือนในระบบ และ audit log

## วิธีติดตั้งและรัน

ต้องมี Node.js เวอร์ชัน 22.5 ขึ้นไป (ใช้ฐานข้อมูล SQLite ที่ติดมากับ Node ไม่ต้องติดตั้งฐานข้อมูลแยก)

```bash
npm install
npm start
```

เปิดเบราว์เซอร์ที่ http://localhost:3000

ฐานข้อมูล (`data.sqlite`) จะถูกสร้างและใส่ข้อมูลตัวอย่างอัตโนมัติเมื่อรันครั้งแรก

## บัญชีทดสอบ (รหัสผ่านทั้งหมด: `123456`)

| รหัส | บทบาท |
|---|---|
| ADMIN001 | ผู้ดูแลระบบ |
| OFF001 | เจ้าหน้าที่ |
| LEC001 | อาจารย์ |
| CON001 | อาจารย์แนะแนว |
| 65010001, 65010002 | นักศึกษา |

บัญชีที่ออกใหม่ผ่านหน้า "ผู้ใช้" จะถูกบังคับเปลี่ยนรหัสผ่านเมื่อ login ครั้งแรก

## สิทธิ์การใช้งานโดยย่อ

- **Admin**: ทุกอย่าง + อนุมัติคำขอเพิ่ม/ลบวิชา + ดู audit log
- **Officer**: ออกบัญชี, ตรวจเอกสาร, เทียบโอน, จัดการหลักสูตร/วิชาเปิดสอน (รอ Admin อนุมัติ), เปิดเทอม, ถอนวิชา
- **Lecturer**: เทียบโอน, ขอเพิ่ม/ลบวิชาเปิดสอน, ถอนวิชา, ตารางสอนที่ตนสอน
- **Counselor**: ดูข้อมูลนักศึกษาอย่างเดียว
- **Student**: อัปโหลดเอกสาร 6 รายการ, ดูหลักสูตร, ลงทะเบียน, ตารางสอน

## Flow สำคัญ

1. นักศึกษาอัปโหลดเอกสาร → Officer/Admin ตรวจ ผ่าน/ตีกลับพร้อมเหตุผล
2. Staff บันทึกเทียบโอน → หน้าหลักสูตรของนักศึกษาขึ้น **ACC** ทันที และห้ามลงเรียนวิชานั้นซ้ำ
3. Lecturer/Officer เพิ่ม-ลบวิชา → รอ Admin อนุมัติ (Admin ทำเองมีผลทันที) มีแจ้งเตือนกระดิ่งในระบบ
4. นักศึกษาติ๊ก ☑ เลือกวิชา → กด "ยืนยันการลงทะเบียน" → ระบบตรวจเวลาชน/ที่นั่งเต็ม/ACC/ลงซ้ำ → หน้าหลักสูตรขึ้น เทอม/ปี → ตารางสอนอัปเดตอัตโนมัติ

## การนำขึ้นออนไลน์ (deploy)

- รองรับ `PORT` จาก environment variable อยู่แล้ว ตั้ง `SESSION_SECRET` เป็นค่าลับของคุณเองด้วย
- **คำเตือนสำคัญ:** โฮสติ้งฟรี (Render/Railway free tier) จะลบไฟล์บนดิสก์เมื่อเครื่อง restart ทำให้ `data.sqlite` และโฟลเดอร์ `uploads/` หาย — การใช้จริงต้องใช้ persistent disk (เช่น Render Starter) หรือย้ายฐานข้อมูลไป PostgreSQL ภายหลัง
- สำรองข้อมูล: copy ไฟล์ `data.sqlite` และโฟลเดอร์ `uploads/` เก็บไว้สม่ำเสมอ

## ต่ออีเมลจริง (ทางเลือก)

ฟังก์ชัน `sendEmail()` ใน `server.js` เป็น stub ที่ log ไว้ — ต่อ SMTP จริงได้โดยติดตั้ง nodemailer แล้วแก้ฟังก์ชันเดียวนี้
